# Anexos finales

## Anexo 1. Checklist breve de revisión antes de entregar

Antes de dar por cerrada una práctica o el integrador, conviene revisar esta lista mínima:

- ¿el HTML valida razonablemente y mantiene una estructura semántica clara?
- ¿los títulos siguen una jerarquía lógica?
- ¿las imágenes relevantes tienen `alt` útil?
- ¿los formularios usan `label` asociado y mensajes comprensibles?
- ¿la interfaz sigue siendo usable en una pantalla pequeña?
- ¿el foco es visible y puede recorrerse con teclado?
- ¿el CSS está ordenado y usa nombres comprensibles?
- ¿el JavaScript evita mezclar demasiadas responsabilidades en una sola función?
- ¿los ejemplos del proyecto integrador se entienden sin depender de explicaciones externas?

## Anexo 2. Glosario mínimo de la cursada

- HTML: lenguaje de marcado que estructura el contenido de una página.
- CSS: lenguaje que define la presentación visual de ese contenido.
- JavaScript: lenguaje que agrega lógica e interacción en el navegador.
- DOM: representación en memoria del documento HTML cargado por el navegador.
- Selector: expresión que permite apuntar a ciertos elementos HTML desde CSS o JavaScript.
- Clase: atributo reutilizable para agrupar elementos bajo un mismo criterio.
- ID: identificador único de un elemento dentro del documento.
- Responsive design: enfoque de diseño que adapta la interfaz a distintos tamaños y contextos de uso.
- Accesibilidad: criterio de calidad orientado a que la interfaz sea usable por más personas y tecnologías.
- Evento: suceso detectado por JavaScript, como un click, un input o un submit.
- Debugging: proceso de observar, detectar y corregir errores o comportamientos inesperados.
- Framework: conjunto de herramientas y convenciones que acelera el desarrollo sobre una base predefinida.

## Anexo 3. Herramientas y recursos generales recomendados

Para seguir practicando después de esta primera etapa, estas herramientas y recursos suelen resultar especialmente útiles:

- Visual Studio Code o un editor equivalente para escribir y organizar código;
- un navegador moderno con herramientas de desarrollo activas;
- MDN como referencia principal para HTML, CSS y JavaScript;
- validadores y revisores como Lighthouse, DevTools y el validador de HTML del W3C;
- repositorios locales o remotos para guardar versiones de las prácticas y del integrador;
- repositorios personales de ejercicios para registrar avances, errores y versiones del integrador.

## Anexo 4. Sugerencias para seguir estudiando

Una vez consolidada esta base, una progresión razonable de temas podría ser:

- consumo de APIs y manejo más explícito de asincronía;
- almacenamiento local y persistencia básica de datos;
- organización modular del código y herramientas de build;
- profundización en accesibilidad y testing de interfaces;
- introducción a frameworks de JavaScript y componentes reutilizables en una etapa posterior.

Ese tramo ya no pertenece a los objetivos centrales de Web 1. La idea de este libro es dejar una base suficientemente clara para llegar a esa siguiente etapa con mejor criterio técnico y menos dependencia de recetas.
